# Next Agent Handoff: Active Architecture

## Product Decision

VibeBase V1 is a local-first, self-hosted Appwrite control plane for founders and coding agents. Dokploy integration and deployment automation are explicitly deferred to V2.

The V1 founder flow is: sign in, receive an automatically provisioned private workspace, reveal one scoped VibeBase agent key, and let an agent create Appwrite-backed database, schema, storage, and app-auth configuration through VibeBase REST or MCP tools.

## V1 Local Architecture

```text
Docker Desktop / local machine
  VibeBase (Next.js control plane and dashboard)
  PostgreSQL (founders, projects, token hashes, audit events)
  Redis (rate limiting and future session support)
  Official self-hosted Appwrite Compose stack
    Appwrite database, storage, workers, and console
```

Use separate containers/services, never a single combined VibeBase/Appwrite container. Docker Compose is the local/hackathon deployment mechanism. The Appwrite compose files are generated by Appwrite's official installer and are not forked by this repository.

## Commands

Credential-free evaluator demo:

    npm install
    npm run local:up
    npm run local:mcp

Real local Appwrite owner setup and demo:

    npm run stack:init
    npm run stack:up
    npm run stack:mcp

`stack:init` generates ignored `.env.local-stack` values and launches Appwrite's official one-time installer. The Appwrite owner then creates the local VibeBase Appwrite project, server API key, and database; those values remain server-side in `.env.local-stack`.

## Security Contract

- Founders and agents receive only VibeBase `vb_` tokens.
- Appwrite admin keys, database credentials, local stack secrets, and `.env.local-stack` never enter browser code, MCP configuration, logs, screenshots, Git, or agent prompts.
- REST and MCP call the same domain service. Every mutating action is scoped, validated, confirmed, and audited.
- Appwrite client endpoint/project/database IDs can be returned to an agent when needed for generated app code; an Appwrite server key cannot.

## V2: Dokploy And Hosted SaaS

Dokploy is not a V1 requirement and should not appear in V1 recordings, evaluator instructions, or product claims. Keep the existing deployment adapter isolated and inactive unless an explicit V2 task reactivates it.

For a hosted beta, VibeBase will run on AWS or another server-capable platform and use VibeBase-owned Appwrite/Dokploy credentials server-side. Customers pay VibeBase; VibeBase manages provider costs, isolation, quotas, and billing. This needs per-tenant resource mapping, secret management, backups, monitoring, and billing before it can be called production SaaS.
